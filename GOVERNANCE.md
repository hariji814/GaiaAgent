# GaiaAgent Governance

## Decision Making

GaiaAgent uses a **consensus-seeking** model:

1. Proposals are discussed in GitHub Issues or Discussions.
2. Maintainers seek consensus among contributors.
3. If consensus cannot be reached, the lead maintainer makes the final call.

## Roles

- **Contributor**: Anyone who submits a PR that gets merged.
- **Maintainer**: Has commit access and merge authority. Responsible for
  reviewing PRs, triaging issues, and guiding the project direction.
- **Lead Maintainer**: Final decision authority on disputed matters.

## How to Become a Maintainer

Maintainers are contributors who have consistently:
- Submitted high-quality PRs
- Reviewed others' PRs constructively
- Participated in design discussions
- Demonstrated good judgment on scope and compatibility

## Versioning

We follow [Semantic Versioning](https://semver.org/):
- **v0.x**: Breaking changes possible between minor versions.
- **v1.0+**: Breaking changes require a major version bump.

The protocol spec is frozen at v0.1. Future protocol versions will go
through a formal RFC process.
